0. 자바 실행 구조 필수

자바는 “클래스 안에서 시작(main)하고, 위에서 아래로 실행”이 기본 흐름이야.

class(클래스)

코드가 들어가는 “틀”. 자바 코드는 무조건 클래스 안에 있어야 함.

public class MyClass {
    // 코드가 여기 들어감
}

main 메소드

프로그램 시작점. 이게 있어야 “실행”이 됨.

public static void main(String[] args) {
    // 여기부터 실행됨
}
파일명 = public class 이름 위 → 아래 순서로 실행

1. 변수 & 자료형 필수

변수는 값을 저장하는 칸, 자료형은 “무슨 값”을 넣을지 정하는 규칙.

변수(variable)

값을 저장하는 공간. 이름(변수명)으로 불러서 사용.

int age = 20;      // 정수
double pi = 3.14;  // 실수
String name = "슬구";

자료형(data type)

대표 5개만 먼저 외우면 됨.

int     : 정수 (예: 10)
double  : 실수 (예: 3.14)
char    : 문자 1개 (예: 'A')
String  : 문자열 (예: "Hello")
boolean : true/false

형변환(casting) 자주 나옴

자료형을 바꾸는 것. 특히 random 정수 만들 때 많이 씀.

int n = (int)3.14;   // 3 (소수점 버림)
double d = 10;       // 10.0 (자동 변환)

2. 연산자 필수

코드에서 “비교/조건”을 만드는 핵심 부품.

비교 연산자

== 같다
!= 다르다
>  크다
<  작다
>= 이상
<= 이하

논리 연산자

&& 그리고(and)  : 둘 다 true여야 true
|| 또는(or)     : 하나라도 true면 true
!  아니다(not)  : true↔false 뒤집기

주의: == vs =

= 는 대입(저장), == 는 비교(같냐?). 초보가 제일 많이 헷갈림.

int a = 10;        // 저장
System.out.println(a == 10); // 비교 결과 출력(true/false)

3. 조건문 & 반복문 필수

조건문은 “갈림길”, 반복문은 “여러 번”. 문제풀이 80%가 여기임.

if (조건문)

if (score >= 60) {
    System.out.println("합격");
} else {
    System.out.println("불합격");
}

for (반복문)

for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

break / continue

break

반복문을 즉시 끝냄(탈출).

for(...) {
    if (조건) break;
}

continue

아래 코드를 건너뛰고 다음 반복으로.

for(...) {
    if (조건) continue;
    // 이 아래는 실행 안 하고 다음 반복
}
중첩 for문 = “서로 비교”할 때 자주 씀 예: 랜덤번호 vs 입력번호 비교

4. 배열(array) 필수

같은 종류의 값을 여러 개 저장. 인덱스는 0부터 시작.

배열 만들기

int[] arr = new int[5];   // 5칸짜리 정수 배열
arr[0] = 10;              // 0번째 칸에 10 저장

배열 출력

System.out.println(Arrays.toString(arr));
// [10, 0, 0, 0, 0]
arr.length = 배열 길이 arr[0] 첫 칸 arr[length-1] 마지막 칸

5. 메소드(method) 중요

코드를 기능별로 묶는 “기능 상자”. main도 메소드야.

메소드 예시

static int add(int a, int b) {
    return a + b;
}

return

메소드 결과를 돌려주고 메소드를 종료함.

return 10;   // 결과 반환 + 종료
함수/메소드 = “기능 묶음” return = 결과 반환

6. 입력/출력 필수

사용자에게 입력 받고, 화면에 출력하는 기본.

출력

System.out.println("한 줄 출력");
System.out.print("줄바꿈 없음");
System.out.printf("%d번째", 1);

입력 (Scanner)

Scanner sc = new Scanner(System.in);
int n = sc.nextInt();      // 정수 입력
String s = sc.next();      // 단어(공백 전까지)
String line = sc.nextLine(); // 한 줄
Scanner는 다 쓰면 close()

7. 랜덤(Math.random) 자주 씀

Math.random()은 0.0 이상 1.0 미만의 실수. 정수 만들려면 곱하고 (int)로 자름.

// 0~9
int n1 = (int)(Math.random() * 10);

// 1~45
int n2 = (int)(Math.random() * 45) + 1;
*범위 공식: (int)(Math.random()*개수) + 시작값

8. 에러 vs 예외 필수

초보가 제일 성장하는 지점. “에러는 나쁜 게 아니라 힌트”야.

컴파일 에러(문법 오류)

빨간 줄. 실행 자체가 안 됨. (괄호/세미콜론/오타)

// 예: ; 빠짐, 괄호 안 닫힘 등

예외(Exception)

실행 중 터지는 문제. (null, 인덱스 범위 초과 등)

// 예: NullPointerException
// 예: ArrayIndexOutOfBoundsException
에러 메시지의 “몇 번째 줄”이 핵심

9. 초보 생존 팁 강추

자바 초보는 “외우기”가 아니라 “패턴 익히기”로 가야 빨리 는다.

1) 코드 흐름부터 보기

입력 → 처리(비교/반복) → 출력. 이 순서가 보이면 끝.

2) 배열은 0부터

인덱스 0부터 시작. 마지막은 length-1.

3) 에러는 “해석 게임”

에러 메시지 + 줄 번호 + 변수 값 확인하면 대부분 잡힘.

4) Copilot은 “보조”

자동완성은 속도는 올려도 이해는 안 올려줌. 설명은 내가 함.
